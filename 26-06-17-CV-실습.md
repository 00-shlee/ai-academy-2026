### 1. 피카츄와 포켓몬볼 이미지 크롤링
이 과정을 거치는 이유는 영상 속 포켓몬과 몬스터볼을 객체 탐지하려면, 먼저 탐지 모델을 학습시켜야 하기 때문
> 즉, **<u>크롤링으로 이미지 수집 -> Roboflow에서 라벨링/데이터셋 구성 -> 모델 학습 -> 영상에 적용해 객체 탐지</u>**

**크롤링을 위해 제공 받은 코드**
```python
# !pip install selenium

# 웹 브라우저를 제어하기 위한 핵심 도구
from selenium import webdriver
# 웹 요소를 찾을 때 '어떤 방법으로' 찾을 지 (선택자)
from selenium.webdriver.common.by import By
# 코드 실행 중간에 잠시 멈추게 하기 위한 도구
import time
# 웹 페이지 이미지 URL에 접속해서 이미지 데이터를 직접 내려받기 위한 도구
import requests
# 운영체제 / 폴더를 만들기 위한 도구
import os
# 텍스트형태의 이미지 인코딩을 위한 도구
import base64

# 크롬 웹 드라이버 실행 (새로운 크롬 창이 열림)
driver = webdriver.Chrome()

# 검색할 키워드 설정
search_query = "피카츄"

# 구글 이미지 검색 URL 생성 (tbm=isch =>  구글 검색에서 이미지 검색을 의미하는 파라미터)
url = f"https://www.google.com/search?tbm=isch&q={search_query}"

# 위 URL로 웹 드라이버(크롬 창)를 이동시킴
driver.get(url)

# 스크롤 충분히 내리기
for _ in range(10):  # _ : 숫자를 변수에 따로 담지 않음
    # 아래로 스크롤 끝까지 내리기 (다음 이미지를 보기 위함)
    driver.execute_script("window.scrollBy(0,document.body.scrollHeight)")
    # 스크롤 후 새로운 이미지가 로딩될 시간 주기 (2초 기다림)
    time.sleep(2)

# class 속성이 YQ4gaf "만" 있는 이미지 선택
thumbnails = driver.find_elements(By.CSS_SELECTOR, "img.YQ4gaf:not([class*=' '])")

# pikachu_images 이름의 폴더 생성 , exist_ok => 폴더가 이미 있으면 에러 발생시키지 않고 그냥 넘어가기
os.makedirs("pikachu_images", exist_ok=True)

count = 0 #저장된 이미지 개수 카운팅
max_images = 400 # 최대 몇 개의 이미지를 저장할지 목표 수량 설정

for img in thumbnails: # 위에서 찾은 모든 썸네일 이미지 요소 하나씩 확인
    if count >= max_images: # 목표 개수 까지만 반복
        break
    try: # 이미지 다운로드 중 에러 발생으로 인한 종료 방지
        src = img.get_attribute("src") # img 태그의 src 속성값 가져오기 (이미지 URL)

        if not src: #src가 비어있을 경우에는 건너뛰고 다음 이미지 보기
            continue

        img_data = None  # 이미지 데이터 저장 변수

        # 1. src가 http로 시작하면 requests로 다운로드
        if src.startswith(('http://', 'https://')):
            img_data = requests.get(src).content

        # 2. src가 data:image로 시작하면 base64로 디코딩
        elif src.startswith('data:image'):
            header, encoded_data = src.split(',', 1) #header(설명) / data 로 분할
            img_data = base64.b64decode(encoded_data) #data를 이미지로 변환(디코딩)

        # 위 두 경우 중 하나라도 해당돼서 img_data가 만들어졌다면 파일로 저장
        if img_data:
            with open(f"pikachu_images/pikachu_{count}.jpg", "wb") as f: #파일만들기(wb: 쓰기모드)
                f.write(img_data) # 이미지 파일 저장

            print(f"{count} 저장 완료")
            count += 1
    except Exception as e:
        print("에러:", e)

driver.quit() #드라이버 종료
print(f"{count}개의 썸네일 이미지를 저장했습니다.")
```
위 코드에서 내가 공부하면 좋을 만한 코드가 있을까?

> 1. 왜 **requests**가 아니라 **Selenium**일까?
- 구글 이미지는 **JavaScript로 그려지고, 스크롤 할 때마다 이미지가 추가로 로딩**되는 동적 페이지(lazy loading)
- 때문에 requests만 쓰면 처음 받은 HTML 뼈대만 가져오기 때문에 이미지의 대부분이 잡히지 않음
- 따라서 브라우저를 띄워서 스크롤까지 시키는 Selenium이 필요
    - 보충 설명
    - **서버가 완성된 HTML을 보내는 경우(정적)**: requests만으로도 충분, JS 실행 필요 없음
    - **JS가 데이터를 채워 넣는 경우(동적)**: 처음 받은 HTML에 데이터가 없고(빈 뼈대), JS가 나중에 채울 때 => Selenium 유리
> **<u>"정적 페이지면 requests + BeautifulSoup, JS로 그려지는 동적 페이지라면 Selenium"</u>**


> 2. **이미지 소스가 두 종류인 이유(http vs data:image)**
- http://...: 평범한 이미지 주소
- data:image/jpeg;base64,...: 이미지 데이터 자체가 텍스트(base64)로 HTML 안에 박혀 있는 경우
- 구글이 작은 썸네일을 base64로 직접 박아두는 이유는 **HTTP 요청 수를 줄여서 페이지를 빨리 띄우기 위해서**

> 3. **css 선택자 트릭:** `img.YQ4gaf:not([class*=' '])`
- img.YQ4gaf: class에 YQ4gaf가 있는 img
- [class*=' ']: class 속성값에 **공백이 들어간 요소** (= 클래스가 여러 개라는 뜻)
- :not(...): 그걸 **제외**
    - 보충 설명
    - **화면에 그려진 이미지만 보면 둘은 완전히 똑같기에** 코드를 들여다보기 전에는 육안으로는 구별이 불가함
    - 따라서 이미지에 **우클릭 후 "이미지 주소 복사"** 후 붙여넣어 보든, **F12(개발자 도구)로 해당 img 태그의 src를 직접 확인**해서 구별
        - base64의 경우에는 src값이 비정상적으로 김; **이미지 데이터를 통째로 글자로 옮겨 적었기 때문**

> **즉, "YQ4gaf 클래스 하나만 가진 img"를 고르는 것**, 실제 결과 썸네일만 잡고 클래스가 여러 개 붙은 UI 아이콘을 걸러내려는 의도

```python
# 학습

model.train(
    data = '/content/drive/MyDrive/Ultralytics/data/pocketmon-1/data.yaml',
    epochs = 80,
    imgsz = 512,
    patience = 10

best_model = YOLO('/content/drive/MyDrive/Ultralytics/runs/detect/train-2/weights/best.pt')

# 그냥 기존 이미지로 확인
best_model('/content/drive/MyDrive/Ultralytics/data/pocketmon-1/train/images/pikachu_100_jpg.rf.3d0a0e2d836347cb89f47fc640a8aa9c.jpg',
           save = True)
```
**위 코드 보충 설명**
- **epochs**: 전체 학습 데이터를 처음부터 끝까지 몇 번 반복 학습할지 **(많을수록 더 배우지만, 과하면 외워버려서 새 이미지에 약해짐 = 과적합)**
> **<u>epochs는 "최대 몇 번까지"의 상한선</u>**

- **imgsz**: 학습에 사용할 이미지 크기(가로·세로 px); **클수록 정확도↑ 속도↓, 작을수록 빠르지만 디테일 손실** (보통 320/512/640 중 선택)

- **patience**: 성능이 N epoch 동안 개선되지 않으면 학습 조기 종료 **(시간 낭비 + 과적합 방지용 안전장치)**
> **<u>patience는 "일찍 멈춰도 돼"의 브레이크</u>**

### 5. 영상 객체 탐지 코드 작성
```python
# openCV 설치하기
!pip install opencv-python

# 모듈 임포트
import cv2

# 모델 불러오기
best_model = YOLO('/content/drive/MyDrive/Ultralytics/runs/detect/train-2/weights/best.pt')

# 영상 불러오기
video_path = '/content/drive/MyDrive/Ultralytics/data/pokemonster.mp4'

# VideoCapture 생성하기
cap = cv2.VideoCapture(video_path)

# 원본 영상 정보 가져오기
fps = cap.get(cv2.CAP_PROP_FPS)
width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))

# 결과 영상 저장 설정
output_path ='./data/detect_result1.mp4'

fourcc = cv2.VideoWriter_fourcc(*'mp4v') # 영상 형식
out = cv2.VideoWriter(output_path, fourcc, fps, (width, height))

# 탐지 및 결과 저장
while cap.isOpened():
    success, frame = cap.read()

    if not success:
        break

    results = best_model(frame)

    annotated_frame = results[0].plot() # [0]에 있는 결과값을 활용하여 라벨링, 바운딩 박스를 그려서 프레임으로 돌려줌

    # 결과 프레임 저장
    out.write(annotated_frame)


cap.release()
out.release()
```
**위 코드 보충 설명**
> **FPS = Frames Per Second, <u>1초에 몇 장의 사진(프레임)이 넘어가는가</u>**
---
```python 
while cap.isOpened():
    success, frame = cap.read()
```
- **while cap.isOpened()**
    - 영상이 열려 있는 동안 계속 반복
    - cap: "cap = cv2.VideoCapture("영상.mp4")"을 통해 그 영상을 쥐고 있음
    - .isOpened: "아직 영상이 정상적으로 열려 있니?" + while => "열려 있는 한 아래를 반복해라"
- **success, frame = cap.read()**
    - 영상에서 사진을 딱 한 장 꺼내는데, 결과는 두 개
    - success: 꺼내기에 성공했는지(T/F)
    - frame: 실제로 꺼낸 사진 한 장
---
```python
if not success:
    break
```
위 **while 문장에서 더 이상 꺼낼 사진이 없으면(영상 끝) 반복을 종료**함
- success가 False가 되기 때문
---
```python
results = best_model(frame)
```
꺼낸 사진 한 장(frame)을 **학습시킨 모델한테 넘겨서 "여기 뭐 있어?"라고 묻는 것**
- best_model: 앞에서 학습한 포켓몬, 몬스터볼 탐지 모델
- 결과로는 **"어디에 무엇이 몇 % 확신으로 있다"**가 results 값에 담김
---
```python
annotated_frame = results[0].plot()
```
0에 있는 **결과값을 활용해서 라벨링, 바운딩 박스를 사진 위에 실제로 그려서** 돌려주는데, annotated_frame은 예를 들어 **"포켓몬 주위에 네모 박스 + 피카츄 0.91 글자가 그려진 사진"**이 됨
- **왜 [0]?**: YOLO는 사진 여러 장을 한꺼번에 처리할 수 있어 결과를 목록으로 돌려줌. 여기선 한 장만 넣었으니 그 목록의 첫 번째(`[0]`)가 사진 결과
---
```python
out.write(annotated_frame)
```
박스 친 사진을 **결과 영상 파일에 한 장씩 추가**함
- out은 위 "out = cv2.VideoWriter(output_path, fourcc, fps, (width, height))"로 만들어 둔 **"새로 저장할 영상"**
- .write()는 out 결과 영상 파일에 한 장씩 이어 붙이고, 이게 반복되면서 박스 친 사진들이 모여 **완성된 결과 영상**이 됨
