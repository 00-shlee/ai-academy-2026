# 자연어 처리
## transformer
### transformer architecture in "Attention is All You Need"
![Transformer Architecture 정리](./images/26-06-19-자연어처리/transformer.jpg)
- 보충 설명
    - transformer: <u>**입력 시퀀스를 받아서 출력 시퀀스를 만드는 구조**</u>
        - transformer의 한 줄 흐름:
    > 입력 문장 → 임베딩+위치 → Encoder(Self-Attention으로 문맥 이해) → Decoder(Masked Attention으로 순서대로 생성 + Encoder 출력 참조) → 확률 분포 → 다음 단어 출력
    - Encoder: 입력을 이해하는 쪽
    - Decoder: 출력을 생성하는 쪽
    - Encoder-Decoder Attention:
    **Q(Decoder)**: "지금 내가 만들려는 단어와 관련된 입력 정보가 뭘까?"
    **K(Encoder)**: 
    "나는" -> 나는 주어 관련 정보, "학생" -> 학생 직업/신분 정보, "이다" -> 이다 서술 정보
    Q와 K를 비교해서 점수를 매기고 이 점수대로 V(실제 정보)를 가져옴
    **V(Encoder)**: "학생"의 실제 의미 정보를 가져옴
        - Decoder: "**지금 내가 뭘 만들고 있는지**"를 잘 알고 있으며, # 질문(Q)
        - Encoder: "**입력 문장이 무슨 뜻인지**"를 잘 알고 있음 # 답변(K,V)

### transformer 핵심 3가지
- Self-Attention: 모든 단어가 다른 모든 단어를 한 번에 직접 참조 → RNN의 장기 의존성 문제 해결 + 병렬 처리 가능 (트랜스포머의 정체성)
- Masking: 미래 단어를 가려서, 학습과 추론의 조건을 일치시킴 (Decoder가 제대로 작동하게 하는 핵심)
- Encoder-Decoder Attention: 입력이랑 출력을 연결하는 다리 (번역, 요약 등에 쓰임)
> Self-Attention으로 이해하고, Masking으로 순서를 지키고, Encoder-Decoder Attention으로 입력과 출력을 연결함.
