# 노인 행동 패턴 및 건강 이상 감지 및 대응 서비스 설계 방향


 ## 1. 보호 대상
| 대상 | 위협 | 보호 방식 |
|------|------|---------|
| 노인 | 고립, 이상, 범죄 | 이상 감지, 데이터 접근 권한 제한 |
| 담당자 | 업무 과중, 누명 | 우선순위 자동화, 행동 기록 |
| 데이터 | 해킹, 유출 | 암호화, 감사 로그 |


 ## 2. 핵심 설계 방향
 ### 2-1. 담당자 이중화
 한 명의 노인 당 최소 두 명의 담당자가 배정되어야 함

 **이유:** 담당자의 근무 태만으로 노인의 이상을 감지하지 못할 수 있음, 따라서 문제 발생 시 해당 담당자들에게 연락이 가는 시스템

 ### 2-2. 담당자에 대한 모니터링은 감시보다는 보호 느낌으로
 해당 담당자에 대한 모니터링과 로깅은 감시가 아닌 보호의 느낌으로 가야 함

 **이유:** 담당자에게 감시 당한 느낌을 주면 시스템에 대한 회피로 이어질 수 있음, 따라서 억울한 민원, 무고, 오해 등에 대한 보호에 대한 개념으로 접근해야 함

 ### 2-3. 예외에 대한 AI와 관리자의 학습
 AI는 실시간으로 수집되는 데이터들을 지속 학습하되, 평균을 벗어난 예외에 대한 데이터는 AI와 사람(관리자) 모두 학습해야 함

 **이유:** AI의 단독 학습은 소용이 없음, 처음 기조가 되는 데이터에 대한 분석과 예외에 대한 건은 사람 또한 학습해야 함

 ### 2-4. 데이터의 정확성
 해당 노인의 행동 패턴 및 건강의 데이터, 혹은 이후 이상의 데이터의 정확도가 매우 높아야 함, 애초에 한 건이라도 사고가 발생하면 신뢰도가 떨어짐

 **이유:** 데이터를 잘못 수집했다거나, 잘못된 데이터로 대응이 잘못 나갔을 때의 기대 위험성이 높음 / 최소 95% 이상을 목표로 해야 할 듯

 ## 3. 데이터의 접근 권한
 ### 3-1. 담당자 접근 가능 데이터
  - 위험도 알림
  - 대응 메시지
  - 담당 노인의 여부

 ### 3-2. 담당자 접근 불가 데이터
  - 노인의 활동 원시 데이터
  - 외출 시간 등의 생활 패턴
  - 타 담당자의 노인 데이터

 ### 3-3. 상위 관리자만 접근 가능한 데이터
  - 담당자 행동 로그
  - 감사 로그 전체

  **이유:** 이웃 뿐만 아니라 해당 데이터를 사용할 수 있는 담당자가 무조건적으로 안전할 것 같지는 않음


  ## 4. 데이터 감사 시스템
  ### 4-1. 기록의 대상
  - 누가, 언제, 어떤 데이터에 접근했는지
  - 데이터에 대한 비정상적인 잦은 조회
  - 직접 방문 없이 데이터만 열람한 경우

  ### 4-2. 방문의 검증
  - 노인 방문 후 이력 입력은 의무화 되어야 함
  - 도어 센서 데이터와 교차 검증
  - 노인 방문 증 갑작스러운 패턴의 멈춤, 미감지, 비명 등 이상 패턴 별도로 저장

  **이유:** 해당 시스템이 범죄의 수단으로 이용되어서는 안 됨

  ### 4-3. 감사 권한
  - 상위 관리자(예. 보건소 직원)
  - 시스템 관리자
