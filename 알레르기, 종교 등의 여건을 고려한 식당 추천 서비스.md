# 알레르기, 종교, 채식 등의 다양한 여건을 고려한 사용자 gps 기반 식당 추천 서비스

## 1. 핵심 설계 방향

### 1-1. 회원가입 및 로그인 탭에 소셜 로그인

사용자의 연령대 및 로그인 이탈 방지를 위해 카카오톡, 네이버, Google, Apple로 소셜 회원가입 및 로그인 가능

- 회원가입 및 로그인 후에는 알레르기 / 종교 / 채식 등 거부 성분 및 음식 체크 해야 함(스킵 가능, 하지만 스킵 시 "아직 알레르기 등 식사 정보 입력이 되지 않았습니다." 배너 작성 전까지 상시 노출)

### 1-2. 다양한 언어의 지원

해외 여행 시 사용이 가능하게 최대한 많은 언어의 지원

- 예를 들어, 한국에 여행 온 무슬림 관광객의 알레르기 카드 번역 시 아랍어 번역이 되지 않으면 불편
- 번역의 질 또한 중요하니, 단계별로 차차 늘리는 게 좋음

### 1-3. 알레르기 카드

알레르기 사용자가 해외여행 시, 식당 종업원에게 보여줄 수 있는 "현지어 + 시각화"된 알레르기 정보 카드

ex) 상황: 도쿄 라멘 가게, 일본어 못 함, 견과류 알레르기 있음

- 면책 문구 필수 표시, LLM을 이용한 번역 기본 제공(프리미엄 결제 시, '검증이 된' 의사 및 약사 검수 시스템)
- → 의사 및 약사 검수 어떻게 할 것?(미해결 과제)

**기존 방식:**

- 구글 번역 켜고 "Do you have nuts in this?" 입력
- 종업원이 못 알아듣거나, 잘못 이해
- 결국 그냥 안 먹거나, 운에 맡김

**응급 카드:**

- 앱 켜고 "응급 카드" 탭 클릭
- 화면에 자동으로 일본어 카드 표시:

```
━━━━━━━━━━━━━━━━━━━━
🚨 食物アレルギーに関する重要なお知らせ

私は以下の食品に重度のアレルギーがあります:

🥜 ナッツ類 (アーモンド、ピーナッツ、
      カシューナッツなど全て)

これらが含まれた料理は
生命に関わる場合があります。

以下の確認をお願いします:
□ 直接の材料に含まれていない
□ 調理油に使用されていない
□ 隣で調理されていない (交差汚染)

ご協力ありがとうございます 🙏
━━━━━━━━━━━━━━━━━━━━
```

만든 이유: 해당 사용자의 식당 도착 시, 종업원에게 2차 검증 가능
+ 번역 내용이 음성 지원이 되어도 좋을 듯?

### 1-4. 사용자의 리뷰

해당 식당을 방문한 사용자의 실제 리뷰(영수증 혹은 결제 내역이 있어야 작성 가능/오정보 및 악의적 이용 방지)를 통해 식당의 정보 업데이트가 필수적으로 주기적으로 이루어져야 함

- 사용자의 영수증 및 결제 내역 검수 시스템 필수(해당 식당의 점주 혹은 사장 보호 기능)
- 해당 식당의 정보가 정확한지 확인 푸시
- 거짓으로 리뷰 작성, 푸시 답변 시, 해당 시스템 경고(3회 경고 시 해당 사용자의 IP 및 가입 정보를 기반으로 서비스 영구 이용 및 재가입 불가)

### 1-5. 사용자의 gps 기반 식당 추천

사용자의 위치 정보를 허용 받아, 사용자가 체크했던 알레르기 등의 리스트를 참고해 식당을 추천함

- gps 기반 근처 식당 정보를 어떻게 제공?(맵 활용해야 하는데 어떻게?; 미해결 과제)

### 1-6. 다인원이 약속해서 특정 장소를 지정 시, 사용자들의 리스트를 모두 고려해 식당을 추천

혼자 이용하는 것도 좋지만, 다인원이 선호하지 않거나 먹지 않는, 먹지 못하는 음식들이 제각각 다르기 때문에
미리 식당을 정할 때 약속 장소를 특정해서 서비스에 지정해 알려 주면, 해당 장소 근처 식당을 제공 받은 리스트들을 기반으로 순서대로 추천함

- 그룹원의 동의 필요 및 임시 그룹 기능을 통해 상시 검색 가능(기본 사용 시, 주 3회 / 프리미엄은 횟수 늘리기)

## 2. 데이터의 접근 권한

### 2-1. 개별 식별 가능 데이터

- 사용자의 알레르기, 종교 등의 민감정보와 기본 개인정보는 철저한 보안 아래 관리되어야 함 (범죄 방지)
- 추천 서비스 제공 외의 다른 용도로 절대 사용되지 않음
- 시스템 관리자만 접근 가능하며, 접근 시 자동 감사 로그 기록
- 접근 사유를 별도로 기록해 2차 검증

### 2-2. 익명화·집계 데이터 (별도 동의 시)

- 개인 식별이 불가능한 통계 데이터는 다음 목적에 한해 활용 가능:
  - 식당 운영자 대상 시장 인사이트 (B2B;business to business 리포트)
  - 정부·지자체 정책 제안
  - 학술 연구 (개인정보 완전 마스킹)
- 사용자는 익명화 데이터 활용 동의를 선택할 수 있으며,
  거부해도 서비스 이용에 제한 없음

### 2-3. 절대 금지 사항

- 개인 식별 가능 데이터의 제3자 제공
- 보험사, 의료기관에 개인 데이터 판매
- 종교·건강 정보의 차별적 용도 활용
- 익명화되지 않은 데이터의 마케팅 활용
