2) 열 단위 주석(’’’, “””): 작은 따옴표나 큰 따옴표 3개로 감싸진 부분은 주석 처리
(여러 줄;multi-line, 블록 주석)
- 원하는 코드 블록 후, ctrl + / 또한 주석 처리
변수: 변할 수 있는 수, 변할 수 있는 정보 / 데이터를 담아두는 상자
x = 10
x = 20
print(x) = 20, x = 10이라는 포스트잇 위에 x = 20이라는 포스트잇을 붙인 것과 같음
따라서, x는 20을 가리킴
예를 들어,
x = 10
y = x
z = y
x = [10, 20]이라고 적고 x를 구하면 [10,20]으로 나오지만 y나 z를 구하면 10이 나옴
이유: y = 10, z = 10을 가리키는 상태에서 x = [10, 20]이라는 포스트잇을 덧붙인 것
따라서 마지막 줄에 있는 x의 값만 변했음
x = [10, 20]
y = x
z = y
x = 100이라는 상태에서 x, y, z를 구하면 당연히 (100, [10, 20], [10, 20])이 나옴
그런데, print(x, y, z)와 x, y, z라고 구했을 때 x, y, z의 값은 각각 같았지만 값의 형태가 다르게 나오는 게 이상했음; 전자는 100 [10, 20] [10, 20]으로 출력이 되고, 후자는 (100, [10, 20],[10, 20])로 나왔음
찾아보니 print()는 여러 개의 값을 받아 화면에 출력하는 함수, print(x, y, z)의 xyz는 print에 전달되는 3개의 인자 즉, 여기에서 “,”는 tuple을 만드는 쉼표가 아닌, 단지 인자를 전달할 때 구분하는 쉼표
 - print 함수는 기본값으로 인자들 사이를 공백 한 칸으로 띄워서 출력하도록 만들어져 있음
 - ex) print(x, y, z) # -> 10 20 30 / print(x, y, z, sep”, “)  # -> 10, 20, 30
 sep(separator, 구분자)라는 옵션으로 사이에 뭘 넣을지 정할 수 있음
그럼 x, y, z로 실행하면 위에 말했듯이 (100, [10, 20],[10, 20])로 출력이 됨
이유: 여러 값을 하나로 묶어 출력해야 하는데, 수정할 필요가 없기 때문에 () 묶여서 나옴
여기에서 []와 ()의 차이: list[]는 수정 가능 tuple()은 수정 불가능
덧붙이자면, 해당 출력값을 나중에 굳이 수정은 할 수 있지만, 지금 당장은 이 순간의 값을 묶어서 보여 주어야 하며 수정을 할 필요도 없기 때문에 ()로 묶어서 보여 주는 것
 - 수정이 불가능해서 tuple로 묶여서 나오는 게 아님
x = [10, 20]
y = x
z = y
x.append(30)에서 z를 구하면 [10, 20, 30]이 출력됨
append(…)는 … 값을 기존 값에 추가하겠다는 의미라 위에 덮어씌우는 느낌이 아님

변수의 문법 규칙
- 영문과 숫자를 사용할 수 있지만, 숫자로 시작하지는 못함
 * syntax error 뜸 / syntax: 구문, 문장 구조
- 특수문자를 사용할 수는 있으나 실무에서 사용하지 않음(언더바(_)는 사용)
- 예약어는 사용하지 않음(if, elif, while, for, etc)
 * if ~ for은 각각 사용처가 따로 있음
- 대소문자는 구분함
- 언더바로만 사용하거나 언더바로 시작할 수 있음
변수의 자료형(고유 표기 기호 있음)
int: 정수형(기호 없음), float: 실수형(소수점; .), str: 문자열(따옴표;””, ‘’), list: 배열로 저장(대괄호), tuple: 변경이 되지 않는 배열로 저장(소괄호), set: 집합 자료형으로 저장(중괄호), 
dict: 사전형으로 저장(키-값 쌍; json과 유사 / 동일X)(중괄호+colon:)
* set {1, 2, 3}, dict{”a”:1, “b”: 2}
** set, list, tuple 비교:
list - 순서 있고, 중복 ok, 수정 ok
tuple - 순서 있고, 중복 ok, 수정 불가
set - 순서 없고, 중복 자동 제거, 수정 가능(추가/삭제만)
 * st = {1, 2, 2, 3} / print(st) = {1, 2, 3}; 중복되는 2 삭제 / st[0] = 오류, 순서 없어서 인덱스로 X
  st.add(4) = {1, 2, 3, 4}
 ** append와 add가 같냐? 의미는 같지만, 쓰는 자료형이 다름(자료형의 성격이 달라서), 
  append는 list(덧붙인다라는 의미, 맨뒤에 붙임), add는 set(추가한다는 의미, 순서가 없으니 위치는 의미 없음; 갑자기 맨앞에 튀어나올 수도 있음; 여기에서 ‘맨앞’도 의미가 없음)
