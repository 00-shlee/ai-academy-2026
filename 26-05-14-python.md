## 변수
- 변할 수 있는 수, 변할 수 있는 정보 / 데이터를 담아두는 상자

```python
x = 10
x = 20
print(x)  # → 20
```
- x = 10이라는 포스트잇 위에 x = 20이라는 포스트잇을 붙인 것과 같음
- 따라서, x는 20을 가리킴

### 예시 1
```python
x = 10
y = x
z = y
x = [10, 20]
```
- x를 구하면 `[10, 20]`이 나오지만 y나 z를 구하면 `10`이 나옴
- 이유: y = 10, z = 10을 가리키는 상태에서 x = [10, 20]이라는 포스트잇을 덧붙인 것
- 따라서 마지막 줄에 있는 x의 값만 변했음

### 예시 2
```python
x = [10, 20]
y = x
z = y
x = 100
```
- x, y, z를 구하면 `(100, [10, 20], [10, 20])`이 나옴
- `print(x, y, z)` → `100 [10, 20] [10, 20]` (공백으로 구분)
- `x, y, z` → `(100, [10, 20], [10, 20])` (tuple로 묶임)

#### 왜 다르게 나오냐?
- `print()`는 여러 개의 값을 받아 화면에 출력하는 함수
- `print(x, y, z)`의 `,`는 tuple을 만드는 쉼표가 아닌, 인자를 구분하는 쉼표
- print 함수는 기본값으로 인자들 사이를 공백 한 칸으로 띄워서 출력

```python
print(x, y, z)           # → 100 [10, 20] [10, 20]
print(x, y, z, sep=", ") # → 100, [10, 20], [10, 20]
```
- `sep(separator, 구분자)`: 사이에 뭘 넣을지 정할 수 있음

#### x, y, z로 실행하면 왜 ()로 묶이냐?
- 여러 값을 하나로 묶어 출력해야 하는데, 수정할 필요가 없기 때문에 ()로 묶여서 나옴
- `[]` vs `()` 차이: `list[]`는 수정 가능, `tuple()`은 수정 불가능
- **수정이 불가능해서 tuple로 묶여서 나오는 게 아님**
- 지금 이 순간의 값을 묶어서 보여줄 뿐

### 예시 3
```python
x = [10, 20]
y = x
z = y
x.append(30)
print(z)  # → [10, 20, 30]
```
- `append()`는 값을 기존에 추가하는 것 → 덮어씌우는 게 아님
- y, z가 x와 같은 상자를 가리키고 있어서 x 내용이 바뀌면 y, z도 함께 바뀜

## 변수의 문법 규칙
- 영문과 숫자를 사용할 수 있지만, 숫자로 시작하지는 못함 (syntax error / syntax: 구문, 문장 구조)
- 특수문자를 사용할 수는 있으나 실무에서 사용하지 않음 (언더바`_`는 사용)
- 예약어는 사용하지 않음 (`if`, `elif`, `while`, `for` 등)
- 대소문자는 구분함
- 언더바로만 사용하거나 언더바로 시작할 수 있음

## 변수의 자료형

| 자료형 | 설명 | 기호 |
|--------|------|------|
| `int` | 정수형 | 없음 |
| `float` | 실수형 (소수점) | `.` |
| `str` | 문자열 | `""`, `''` |
| `list` | 배열로 저장 | `[]` |
| `tuple` | 변경 불가 배열 | `()` |
| `set` | 집합 자료형 | `{}` |
| `dict` | 사전형 (키-값 쌍, json과 유사/동일X) | `{key: value}` |

### list, tuple, set 비교

| | list | tuple | set |
|--|------|-------|-----|
| 순서 | 있음 | 있음 | 없음 |
| 중복 | 가능 | 가능 | 자동 제거 |
| 수정 | 가능 | 불가 | 가능 (추가/삭제만) |

```python
# set 예시
st = {1, 2, 2, 3}
print(st)  # → {1, 2, 3} (중복 2 삭제)
st[0]      # → 오류 (순서 없어서 인덱스 X)
st.add(4)  # → {1, 2, 3, 4}
```

### append vs add
- 의미는 같지만 쓰는 자료형이 다름
- `append` → list (맨 뒤에 덧붙임)
- `add` → set (순서가 없으니 위치는 의미 없음)

## 변수명 권장 사항

### 1. snake_case 사용 권장
```python
user_name = "승현"  # 권장
userName = "승현"   # python에서는 잘 사용하지 않음 (js 스타일)
```

| 표기법 | 설명 |
|--------|------|
| `snake_case` | 단어 사이를 언더바로 연결, 뱀처럼 길게 이어짐 |
| `camelCase` | 중간에 대문자가 낙타 혹처럼 튀어나옴 |
| `PascalCase` | 파스칼 언어에서 유래, 첫 글자도 대문자 |
| `kebab-case` | 꼬치(kebab)에 꽂힌 모양 |
