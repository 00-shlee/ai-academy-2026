# 컴퓨터 비전
## 1. 이미지 처리와 컴퓨터 비전
- 아날로그 이미지 : 현실 세계의 연속적인 신호로 표현되는 이미지 (필름 사진, 아날로그 CCTV 영상 등 )
- 디지털 이미지 : 픽셀과 숫자로 표현되는 이미지 (스마트폰 사진, 웹 이미지)

### 이미지 처리
- **아날로그 이미지 처리**
    - 이미지 자체를 직접 조작하는 것이 아니라 빛이나 전기 신호를 조절하여 컴퓨터 없이 물리적으로 처리하는 방식 (카메라 렌즈에 색상 필터 부착, 필름 인화 과정에서 밝기 조절, 아날로그 TV 영상 보정 등)
- **디지털 이미지 처리**
    -이미지를 숫자로 변환한 후 컴퓨터로 처리하는 방식 (흑백 변환, 이미지 크기 변경, 블러 처리, 윤곽선 검출 등)

---
```
흑백 이미지(2차원): 각 픽셀이 오로지 밝기(밝음 ↔ 어두움) 하나의 값만 가지기 때문에 단순히 2차원 배열(행렬)으로 표현 가능

컬러 이미지(3차원): 각 픽셀이 RGB, 세 가지 색상 채널의 값을 각각 가지기에 픽셀 하나를 표현하는 데 숫자 3개가 필요함
=> 이미지 전체가 (세로) × (가로) × (채널 수, 즉 3)` 형태의 3차원 배열 형태가 됨
```
---

- **디지털 이미지 처리의 활용 분야**
    - 의료 분야, 자율 주행 및 교통, 보안 및 감시, 제조 및 산업, OCR 및 문서 처리, 스마트폰 및 SNS, 위성 및 드론, AI 및 컴퓨터 비전 등

- **이미지 처리와 컴퓨터 비전의 관계**
- 이미지 처리는 컴퓨터가 이미지를 보다 효율적으로 활용할 수 있도록 이미지를 개선하거나 변환하는 기술을 의미
    - 예시) 흐릿한 사진을 선명하게 만들거나, 노이즈를 제거하거나, 이미지의 크기를 조정하는 작업 등

- 컴퓨터 비전은 이미지 속에 담긴 정보를 분석하고 이해하여 의미 있는 결과를 도출하는 기술
    - 예시) 이미지 속 객체를 인식하거나 분류하고, 사람의 얼굴을 식별하거나, 자율 주행 자동차가 도로 상황을 판단하는 작업 등

- **디지털 이미지 처리 단계 (이미지 획득 → 이미지 개선 → 이미지 분석 → 이미지 해석 및 이해)**

### 컴퓨터 비전
컴퓨터가 이미지와 영상과 같은 시각 정보를 사람처럼 인식하고 이해하며 해석할 수 있도록 하는 인공지능 기술 분야

⇒ 디지털 이미지를 통해 **우리가 세상을 인식하고 이해하는 방식을 모방하고, 이미지로부터 의미 있는 정보를 추출하는 것**

- 컴퓨터 비전 작업 수준 별 분류
    - 낮은 수준 : 이미지 자체의 품질을 개선하거나 기본적인 특징을 추출하는 단계
        - 노이즈 제거, 밝기 및 대비 향상, 채도 및 색상 보정, 엣지 검출, 이미지 필터링 및 선명화

    - "낮은 수준의 컴퓨터 비전”과 “이미지 처리”를 헷갈릴 수 있음
        - 이미지 처리가 컴퓨터 비전에 포함된다고 보기 어려운 이유가 각각 사용하는 “목적”에 차이가 있기 때문
        - “노이즈 제거”를 예로 들면 이미지 처리는 ‘사진을 깨끗하게 만드는 것’ 자체가 목적이며, 컴퓨터 비전은 이후 물체를 인식하거나 분류하기 위한 준비에 해당하는 전처리가 목적에 해당함
        - 따라서 **두 분야를 포함 관계로 이해하기 보다는 교집합이 있는 별개의 분야로 이해하는 것이 좋음**

    - 중간 수준 : 이미지 내의 구조와 객체를 구분하고 특징을 분석하는 단계
        - 이미지 분할, 객체 영역 분리, 특징 추출, 광학 흐름(이동 방향과 크기) 추정, 객체 추적

    - 높은 수준 : 이미지의 의미를 이해하고 판단하는 단계
        - 객체 인식 및 분류, 얼굴 인식, 장면 이해 및 재구성, 행동 인식, 이미지 기반 추론 및 의사 결정

---

=> **컴퓨터 비전의 모든 단계**는 이미지 속에서 의미 있는 정보를 찾아내는 **특징 추출(Feature Extraction)을 기반**으로 함 (특징 : 엣지, 모서리, 색상, 질감, 형태, 패턴 등)

과거에는 사람이 직접 특징을 설계하는 방식(SIFT, SURF, HOG 등)을 사용했지만, 최근에는 **합성곱 신경망(CNN, Convolutional Neural Network)이 이미지로부터 특징을 자동으로 학습하여 추출**

**즉, 컴퓨터 비전의 핵심은 이미지로부터 유용한 특징을 추출하는 것이며, 현대 컴퓨터 비전에서는 CNN이 이러한 특징 추출의 핵심 기술로 활용되고 있음**

## 2. 이미지 이해하기
### 디지털 이미지의 구조
- 픽셀(pixel): Picture(그림)와 Element(요소)의 합성어로, 디지털 이미지를 구성하는 가장 작은 단위
    - 흑백 이미지 - 밝기(Intensity) 값만 저장
    - 컬러 이미지 - 빨강(Red), 초록(Green), 파랑(Blue) 색상 값을 저장)

- 해상도(Resolution): 이미지가 가로와 세로 방향으로 몇 개의 픽셀로 구성되어 있는지를 의미
    - 일반적으로 가로  x  세로 형식으로 표현

    - 해상도가 높을 수록 더 많은 정보를 표현할 수 있어 이미지가 선명해지며, 카메라 센서의 성능, 스캐너의 정밀도 등이 해상도에 영향을 줌

- 픽셀 밀도(Pixel Density) : 일정한 화면 크기 안에 픽셀이 얼마나 촘촘하게 배치되어 있는지 나타내는 값
    - 보통 PPI (Pixels Per Inch) 단위를 사용

    - 픽셀 밀도가 높을 수록 개별 픽셀이 눈에 잘 보이지 않아 이미지가 더 부드럽고 선명해 보이며 텍스트와 그래픽의 가독성이 향상
    
    - 스마트폰 : 약 300~500 PPI

    - 모니터 : 약 90~200 PPI
    
    →  같은 해상도라도 화면 크기가 작을 수록 픽셀 밀도는 높아짐
    

- 서브 픽셀 (Subpixel) : 하나의 픽셀을 이루는 더 작은 색상 요소
    
    - 일반적인 디스플레이의 한 픽셀은 Red(R), Green(G), Blue(B) 세가지 서브 픽셀로 구성

    - 각 서브 픽셀의 밝기를 조절하여 다양한 색상을 표현
    
    - RGB (255, 0, 0) → 빨간색 / RGB (0, 0, 0) → 검은색

### 이미지 압축(Image Compression)
**이미지는 파일 크기가 크기 때문에 저장 공간을 절약하고 전송 속도를 높이기 위해 압축 과정을 거치며**, 압축 방식은 크게 무손실 압축(Lossless Compression)과 손실 압축(Lossy Compression)으로 나뉘어짐

---
- **무손실 압축**: 데이터를 압축한 후 다시 복원했을 때 원본 이미지와 완전히 동일한 데이터를 유지하는 방식
    - 활용 사례 : 로고 이미지, 아이콘, 문서 스캔본, 의료 영상, 이미지 편집 작업용 원본 파일
    - 대표 포맷 : PNG, GIF

- **손실 압축**: 사람의 눈으로 구분하기 어려운 정보를 제거하여 파일 크기를 줄이는 방식
    - 활용 사례 : 웹사이트 이미지, SNS 업로드 이미지, 스마트폰 사진, 온라인 쇼핑몰 상품 이미지
    - 대표 포맷 : JPEG(JPG)
    ---

### 색 공간 (Color Space)
**색상을 숫자로 표현하기 위한 체계(규칙)**

색 공간이 필요한 이유: 같은 빨간색이라도 목적에 따라 표현하는 방식이 다르기 때문

---
- RGB : 가장 많이 사용되는 색 공간으로, 빛의 3원색을 이용하여 색을 표현

    - R(Red), G(Green), B(Blue) → 일반적으로 0 ~ 255 범위를 가짐

        - 색상 분리 작업이 어려움
        - 디지털 이미지와 모니터에서 가장 많이 사용
        - 사람에게는 직관적이지 않을 수 있음
        - ex) 모니터, 스마트폰, 카메라 (화면 표시)

 ---   
- HSV : 사람이 색을 인식하는 방식과 유사하여 컴퓨터 비전에서 자주 사용

    - 색상(Hue), 채도(Saturation), 명도(Value)로 색을 표현하는 방식

        -  특정 색상 추출이 쉬움
        - 조명 변화에 상대적으로 강함
        - 객체 검출 및 색상 분할에 많이 사용
        - ex) 신호등 인식, 피부색 검출, 특정 색상의 물체 추적 (색상 검출)
---
- **Grayscale : 색상 정보 없이 밝기 정보만 사용하는 방식**
    - 한 개의 값만 사용
    - 계산량 감소
    - 이미지 처리 속도 향상
    - ex) OCR 문자 인식, 엣지 검출, 의료 영상 분석 (밝기 분석)
---
- **CMYK : 프린터에서 사용하는 색 공간**

    - C(Cyan), M(Magenta), Y(Yellow), K(Black) 으로 색을 표현하는 방식
    
        - 잉크 기반 색상 표현
        - 인쇄물 제작에 사용
        - RGB와 표현 가능한 색상 범위가 다름
        - ex) 책/잡지 인쇄, 포스터 제작, 광고물 출력 (인쇄)

# THRESH_TRUNC: 임계값보다 큰 픽셀값은 모두 임계값, 아니면 변경하지 않음 => 이미지 밝기 조절에 사용하여 세부 정보가 잘 드러나게

# (임계값, 변형된 이미지); 하나만 필요하면 a,b 중 필요하지 않은 부분을 '_'로 표현
_, thr_ryan = cv2.threshold(ryan, 100, 255, cv2.THRESH_BINARY)

# OTSU => 가장 적절하게 이진화 할 수 있는 임계값을 찾아줌
# => 히스토그램
thres, thr_ryan2 = cv2.threshold(ryan, 100, 255, cv2.THRESH_BINARY | cv2.THRESH_OTSU)

# 행, 열 지정해서 여러 개의 그래프, 이미지를 출력
# (행, 열, 순서)
plt.subplot(1, 3, 1)
# 축 없애기
plt.axis('off')
plt.imshow(ryan, cmap = 'gray')

plt.subplot(1, 3, 2)
plt.imshow(thr_ryan, cmap = 'gray')

plt.subplot(1, 3, 3)
plt.title(thres)
plt.imshow(thr_ryan2, cmap = 'gray')
    
plt.show()
